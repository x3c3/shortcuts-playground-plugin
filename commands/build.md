Delegate to the `shortcut-builder` agent (via the Agent tool with `subagent_type: shortcut-builder`) to design, build, validate, sign, and archive a macOS/iOS Shortcut that satisfies the following brief:

$ARGUMENTS

The agent owns the full Craig Loop. It must:

1. Read `skills/shortcuts-playground/SKILL.md` plus any relevant reference files it needs for the specific actions in play (`BEST_PRACTICES.md`, `ACTIONS.md`, `APPINTENTS.md`, `VARIABLES.md`, `CONTROL_FLOW.md`, `FILTERS.md`, `PARAMETER_TYPES.md`, `EXAMPLES.md`).
2. Pick an icon and color by running `resolve-icon --prompt "<brief>"` (plus `--icon`/`--color` overrides if the user specified them in the brief).
3. Generate the plist XML with the two required leading Comment actions (the second containing the mandatory `Shortcuts generated by Shortcuts Playground. May contain mistakes.` disclaimer).
4. Write the draft to `${CLAUDE_PLUGIN_OPTION_OUTPUT_DIR:-$HOME/Documents/Shortcuts Playground}/drafts/<name>.xml`. The `PostToolUse` hook will auto-run `validate-shortcut` on the write.
5. Run the Craig Loop: read validator errors, make targeted edits, re-validate. Max 5 iterations. Stop and ask the user if the same error persists across two consecutive iterations.
6. Once validation passes, run `sign-shortcut <draft.xml> --name "<final name>"` to archive the unsigned XML and produce a signed `.shortcut`.
7. Report: the signed `.shortcut` absolute path, the archive XML absolute path, a one-sentence summary of what the shortcut does, and any caveats the user needs to know (required permissions, API keys, runtime prerequisites).

Do not execute the shortcut or import it into Shortcuts.app — the user will do that themselves by opening the signed file.
